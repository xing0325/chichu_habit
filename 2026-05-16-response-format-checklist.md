# 2026-05-16 每次回答前列任务待办清单（chichu 的硬规定）

这是 chichu 对回答格式的**硬规定**，所有进来的 AI session 必须遵守。原版来自他的 memory（`feedback_response_format_checklist.md`，originSessionId `0ec6d790-dea6-4d8c-812c-a5a1e893600e`），今天搬一份到这里方便所有 session 第一手看到。

## 规则

**每次回答开头必须列一个任务待办清单**，格式如下：

````
## 待办清单
- [x] 已完成的事项 🤖（纯 AI）
- [ ] 未完成的事项 👤（必须人肉）
- [ ] 未完成的事项 🤝（AI+人肉混合）
````

## 符号约定

- `[x]` = 已完成（打勾）
- `[ ]` = 未完成（相当于打叉）
- 🤖 = 纯 AI 可做
- 👤 = 必须人肉做（chichu 亲自，浏览器交互/扫码/输密码/物理操作）
- 🤝 = AI + 人肉混合（AI 做主体，chichu 配合 / 审核 / 操作 UI / 提供 token）

## 清单维护规则

- 这个清单**不是固定的** — 根据对话进展持续增加新任务
- **谨慎减少** — 完成的不要删，打勾保留
- 通常**只增加、只打勾，几乎不删除**
- **每次回答都要重新展示完整清单**（保留历史已完成项）

## 为什么 chichu 要这条

他希望随时看到任务全貌、谁要做什么、进度到哪了。**符号化的标签让他扫一眼就知道哪些是自己的活儿，不用读完整段文字**。

这条 chichu 后来还专门挑出来要分享给同学（见 2026-05-11 那条），说"分 AI 和人肉的方法非常有用、一目了然"。证明他对这条很看重，**别图省事跳过**。

## 怎么落地

- 收到任何用户消息（包括澄清问题、状态查询、闲聊）→ 回答最前面先列清单
- 当前会话的 brainstorming 任务也要纳入清单
- 不确定一条任务是 🤖 还是 🤝？倾向标 🤝 —— 让 chichu 主动说"这个你自己干就行"再降为 🤖

## 反模式

- 用文字"AI / 人肉 / 混合"代替 🤖 / 👤 / 🤝（emoji 是 chichu 钦定的，文字版没那么醒目）
- 中途某条回答漏列清单 —— 哪怕是一句话回答也要带
- 完成的事项删掉而不是打勾 —— 历史进度会丢
- 把清单放在回答末尾而不是开头 —— 失去"扫一眼"的意义
