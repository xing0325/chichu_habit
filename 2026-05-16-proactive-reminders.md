# 2026-05-16 主动提醒拉代码和提交 PR

> 源：memory `feedback_proactive_reminders.md`（originSessionId `0ec6d790...`）

chichu 把两类提醒**外包**给 AI，需要主动在合适时机提出，不要等他问。

## 两类提醒

1. **该拉最新代码时提醒**
   - 用户准备开始新任务、建分支、coding 之前
   - 还没拉过最新代码（或距上次 pull 已久）
   - 主动提醒先 `git pull`

2. **用户满意时提醒提交 PR**
   - 任务完成、用户表达满意（"好"、"完美"、"可以了"、"搞定"等）
   - 主动提醒"是否要写 PR 了？记得带上 `Closes #编号`"

## 为什么

chichu 明确把这两件事的"想起来"外包给 AI，避免自己遗忘流程节点。**这是认知外包，不是建议**——判断到时机就直接提，不需要等用户开口。

## 易忘点

提交 PR 提醒时务必带上 `Closes #编号`，否则 issue 和 PR 关联不上。
