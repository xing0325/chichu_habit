# 2026-05-16 美工需求交给 Codex + image-2，别用 emoji 凑数

chichu 买了 Codex 会员，跟 Claude Code 有了明确分工。这一条留给以后所有进来的 AI session 参考。

## 分工

- **Claude Code 干**：文字、代码、结构、清单、文档
- **Codex 干**：图（用 image-2 生成）

## 规则

涉及**美工 / 视觉 / 平面设计**需求时（logo、海报、装饰图、配图、示意图、icon、banner、封面……），**不要用 emoji 拼凑或 ASCII art 凑数假装做了视觉**。

正确做法：**告诉 chichu 去 Codex 用 image-2 接手这一步**，并把图的描述（prompt）整理清楚交给他拿走。

## 触发场景

- 给文章 / 分享内容配图
- 需要 icon / logo / 装饰元素
- 需要示意图、海报、banner、封面
- "美化"页面 / 文档的视觉表达
- chichu 提到"做个图"、"配张图"、"美工"、"设计一下"

## 不触发的场景

- **功能性 emoji 标签**（todo 清单的 🤖 / 👤 / 🤝 这种）— chichu 明确要的格式，继续保留
- 纯文字 / 代码 / 表格能完成的产出
- 没人提视觉需求，只是写功能 / 文档

## 别做的事

- 用一堆 emoji（🎨🌈✨💫）伪装"我做了视觉"
- 硬凹 ASCII art 充当图
- 主动提议"我加张截图风格示意图"之类的自我装饰
- 在分享给同学 / 对外的内容里偷懒堆 emoji 当装饰

## 怎么落地

碰到美工需求时：

1. 把文字 / 结构这块做完
2. 写一段清楚的图片 prompt（要什么风格、几张、画面元素、用途）
3. 告诉 chichu："这部分图交给 Codex 用 image-2 出，prompt 我给你写好了：……"
4. 不要自己硬上 emoji 或 ASCII

## 为什么

chichu 买了 Codex 会员就是为了拿真图，不是让 Claude Code 用 emoji 拼贴对付。专业分工，对外发的东西才好看。
