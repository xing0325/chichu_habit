# 2026-05-20 讲解 / 教学 HTML 要图片多文字少

> 源：memory `feedback_explain_html_visual_heavy.md`

写"教学 / 讲解 / 课程"类 HTML 时，**视觉优先 + 文字精简**。

## 规则

- 概念用**图 / 示意 / SVG 简笔**表达，不用大段文字解释
- 设备 / 零件 / 选项一律用**卡片网格**（icon + 名称 + 1 行说明 + 标签 chips），不用纯表格
- 数据统计用**条形图 / 柱状图 SVG**，不用数字表
- 时间线 / 演化用**横向 timeline SVG**，不用文字段落
- 每个概念配示意图：交互区域图 / 力曲线 / 剖视图等

## 为什么

chichu 在 vibe-pen `lessons.html` 项目里明确说"**html 更直观，多一点图片少一点文字**"。

他的学习路径偏视觉。

## 怎么落地

- 表格能换成卡片网格就换，每张卡片包含 SVG 缩略图
- 长文字段落能拆成 callout 就拆，能换成图就换
- 用色块徽章 / 标签 chip 代替"分类 1 / 分类 2"的文字
- 但**仍然不要用 emoji 凑数**（参考"美工需求交给 Codex + image-2"那条）—— SVG 简笔示意图是合法的工程沟通图，不是美工需求
- 真要"美图 / 海报级配图"时仍走 Codex + image-2

## 边界

| 场景 | 怎么处理 |
|---|---|
| 工程示意图、交互区域、零件结构 | inline SVG 自己画 |
| 数据可视化（条形图、柱状图、timeline） | inline SVG 自己画 |
| icon / 标签徽章 | 色块 + 文字，或简笔 SVG |
| 海报、封面、illustration | 推 Codex + image-2 |
