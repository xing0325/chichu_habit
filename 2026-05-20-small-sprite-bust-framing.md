# 2026-05-20 小尺寸 sprite 优先半身 / 胸像构图，可破画框

> 源：memory `feedback_small_sprite_use_bust_framing.md`

为桌宠 / sprite / icon 写视觉 brief 时，**默认建议半身（胸像或腰以上），不要全身像**。同时主动提议**非正方画布**和**"打破第四面墙"**（角色或元素越出 canvas 边）作为表现力放大器。

## 为什么

bypass-pet 2026-05-18 v1 让 Codex 出 128×128 全身像，结果脸只占 ~60px，五官完全糊。

chichu 原话："他生成的是全身像，而这个不太适合当桌宠"。

第二版改成 256×320 浴室镜子胸像 + Tyler 的手伸出镜面越出 canvas 右边，立刻读得出角色。"窗户/镜框作为画框 + 角色越出边缘" 是这个尺寸最有效的叙事杠杆。

## 怎么落地

- 角色类 sprite 在 **≤ 256px** 时，brief 里**写死**："bust / waist-up framing"，禁止全身
- 256-512px 区间允许半身到全身的选择，但**仍偏向半身**
- 主动提议：画布可以**非正方**（256×320、224×288 这种竖向比例往往更合适，给角色头部更多空间）
- 主动提议：**四面墙破壁元素**（手 / 烟 / 旗 / 光线越出 canvas 边）能让小尺寸 sprite 有"突破容器"的张力，特别适合双态对比设计
- 若 chichu 已经指定了某种构图（如"窗户"），优先围绕他的想法放大，**不要切换到 AI 自己的备选**
