# 2026-05-16 输入器件优先机械结构而非电容触控

> 源：memory `feedback_prefer_mechanical_inputs.md`（originSessionId `19e87b41...`）

为硬件 / 外设设计输入方式时，按 **"机械结构 > 电容触控"** 的顺序推荐。

- **优先列**：机械旋钮、滚轮、扳机、摇杆、拨杆、按键
- **降权列**：电容触控环、触控条、压感、敲击 —— 只在机械方案明显更糟时才列

## 为什么

chichu 主观感受电容触控**不太灵敏**，更喜欢有反馈、有阻尼、有咔哒声的机械手感。但他**不一禁触控**——明确说过"真的好的话我会采用"，所以触控不是黑名单，只是降权。

## 怎么落地

- 列方案时机械放前面、touch 放后面
- 标明触控的得失（如"零件少 + 静音 + 无机械磨损"）让 chichu 自己判断
- 同等手感时**优先选机械**，不要因为"触控更现代"而推荐
- 出现真正机械方案做不到的场景（如 360° 滑动 + 多指 + 极薄空间）才以触控为首选，并解释为什么这次触控胜出
