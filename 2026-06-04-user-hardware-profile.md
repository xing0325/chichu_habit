# 主力机硬件配置(回答"我电脑能跑 X 吗"先看这)

> 源：memory `user_hardware_profile.md`

chichu 的主力机是一台 Windows 笔记本。以后他问"我电脑能不能跑某个项目/某个本地模型"，先对照这份配置判断，再决定要不要实测，别每次重新查机器。

## 配置(截至 2026-06)

- **CPU**：Intel Core Ultra 7 155H(Meteor Lake，16 核 22 线程)——很强
- **内存**：31.5 GB——充足
- **显卡**：只有 Intel Arc 核显(集显)，**没有 NVIDIA 独立显卡，没有 CUDA**
- **系统**：Windows 11 家庭中文版(10.0.26200，64 位)
- **磁盘**：C 盘剩 ~136 GB
- **接口**：有 USB4 / 雷电4 口

## 判断兼容性的几条硬约束

1. **没有 NVIDIA GPU** → 需要 CUDA 的本地大模型推理或训练，要么跑不动(只能退 CPU，很慢)，要么直接不支持。
2. **不是 Apple Silicon** → 基于 Apple MLX 框架的项目(例如明略的 Mano-P)本地跑不了，只能走云端 API 或外接算力棒。
3. **是 Windows 不是 macOS/Linux** → 官方只给 `brew` 或原生 Linux 安装路径的项目，得用 WSL 或找替代装法。
4. **有 USB4/雷电4** → 需要外接算力棒/eGPU 这类方案时，物理接口是具备的(但要额外买硬件)。

## 由来

2026-06-04 chichu 问"去看一下 Mano-P 这个项目，我的电脑能用吗"。查下来 Mano-P 的本地模式基于 MLX、硬性要求 Apple M4 Mac + 32GB 内存，这台 Intel + Windows 机器本地跑不了(只能走云端模式或买算力棒)。借这次把机器配置固化下来，避免以后重复探测。
